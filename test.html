Standardize common font fallbacks

The font on this page is set by clear.css to Arial, Helvetica, sans-serif which are common sans-serif font fallbacks. We used sans-serif because it appears to be the most common font-type on the web to use. Resetting to this common font fallback also prevents developers including the fallbacks in each font-family property they use.

Set root font-size to 62.5% of the browser standard.

Using font-size: 62.5% on the html element should give us the value of 10px; trusting that browsers have the default value of 16px. By doing this we can make REM/EM calculations easier e.g. 20px after this change would be 2rem

Although we default the html element to 10px, we also added a font-size of 1.6rem to the body element. This means that you still use the standard font-size of 16px (1.6 * 10 = 16) but with the addition of easier REM calculations and a REM unit font-size on the body as standard. How cool is that?

Minimal link styling

If you're a developer you must of, at one point, styled an anchor element. If you have, you've probably also removed the default link underline so that you may then extend the styling of the link. Clear.css has the underline removed by default to that we developers can have extension styling in our CSS rather than a reset and extension in one block.
